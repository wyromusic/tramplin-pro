# AGENT: tramplin.pro — Школа прыжков на батуте

## Контекст проекта

**Сайт:** tramplin.pro  
**Тип:** Веб-платформа для управления школой батутного спорта  
**Язык интерфейса:** Русский  
**Валюта:** Рубли (RUB)  

---

## Стек технологий

### Frontend
- **Next.js 14** (App Router) — основной фреймворк
- **TypeScript** — обязательно везде
- **Tailwind CSS** + **shadcn/ui** — UI компоненты
- **React Query (TanStack Query)** — серверный стейт и кэш
- **Zustand** — клиентский стейт (корзина, UI)
- **React Hook Form + Zod** — формы и валидация

### Backend
- **Next.js API Routes** (или отдельный **Fastify** если понадобится)
- **Supabase** — база данных (PostgreSQL), auth, realtime, storage
- **Supabase Auth** — аутентификация и управление ролями через RLS

### Интеграции
- **ЮKassa** — приём онлайн-платежей
- **SMS.ru или SMSC.ru** — SMS уведомления
- **Web Push (VAPID)** — push-уведомления в браузере
- **Vercel** — деплой frontend
- **Supabase Cloud** — БД и backend

---

## Роли пользователей

| Роль | Описание |
|------|----------|
| `student` | Клиент/студент/арендатор. Записывается, платит, получает уведомления |
| `teacher` | Преподаватель. Ведёт расписание, студентов, отмечает посещаемость |
| `admin` | Полный доступ. Видит всё: финансы, переносы, аналитику |

Роли хранятся в Supabase через `user_metadata` + Row Level Security (RLS).

---

## Основные сущности системы

- **User** — пользователь (студент / преподаватель / админ)
- **Teacher** — профиль преподавателя
- **Student** — профиль студента (привязан к User)
- **Course** — курс (набор занятий по программе)
- **LessonType** — тип занятия: вводное / групповое / индивидуальное / самопрактика / аренда
- **Slot** — временной слот (расписание доступности преподавателя или зала)
- **Booking** — запись студента на конкретный слот
- **Purchase** — покупка (курс или пакет часов)
- **Payment** — платёж через ЮKassa
- **Notification** — уведомление (SMS или Push)

## Ключевые пользовательские сценарии

### Студент
1. Регистрация / вход
2. Запись на вводный урок (без оплаты или с предоплатой)
3. Запись на обычный урок (списывается с баланса или курса)
4. Покупка курса / пакета часов — оплата через ЮKassa
5. Запись на самостоятельную практику / аренду зала
6. Просмотр своих записей и истории
7. Получение SMS/Push за 24ч и за 2ч до урока
8. Перенос или отмена записи

### Преподаватель
1. Настройка расписания доступности (по неделям)
2. Просмотр своих предстоящих уроков
3. Список своих студентов и их прогресс
4. Отметка посещаемости
5. Инициирование переноса урока

### Администратор
1. Дашборд: занятость, выручка, посещаемость
2. Управление расписанием всей школы
3. Управление переносами и отменами
4. Управление курсами и ценами
5. Управление пользователями
6. Финансовые отчёты (ЮKassa + ручные платежи)
7. Рассылка уведомлений

## Правила разработки

1. **TypeScript strict mode** — никакого `any`
2. **RLS в Supabase** — безопасность на уровне БД, не только на фронте
3. **Zod схемы** — валидация на фронте и в API
4. **Транзакции** — любое изменение баланса/записи — только через транзакцию
5. **Идемпотентность платежей** — использовать `idempotency_key` в ЮKassa
6. **Mobile-first** — интерфейс должен работать на телефоне прежде всего
7. **Русский язык** — все сообщения об ошибках, уведомления, UI — на русском

## Структура проекта (Next.js App Router)

```
src/
├── app/
│   ├── (public)/          # Лендинг, запись без авторизации
│   ├── (student)/         # Личный кабинет студента
│   ├── (teacher)/         # Кабинет преподавателя
│   ├── (admin)/           # Админ-панель

├── components/
│   ├── ui/                # shadcn/ui компоненты
│   ├── booking/           # Компоненты записи
│   ├── calendar/          # Компоненты расписания

├── lib/
│   ├── supabase/          # Клиент, типы, хелперы
│   ├── yukassa/           # ЮKassa интеграция
│   ├── notifications/     # SMS + Push
│   └── validations/       # Zod схемы
└── types/                 # Глобальные TypeScript типы
```
