# AGENT: tramplin.pro — Школа прыжков на батуте

## Контекст проекта

**Сайт:** tramplin.pro  
**Тип:** Веб-платформа для управления школой батутного спорта  
**Язык интерфейса:** Русский  
**Валюта:** Рубли (RUB)  
**Аудитория:** Россия. Всё должно работать без VPN, без иностранных сервисов.

---

## ⚠️ Требование: работа в России без блокировок

Все сервисы — только российские или с надёжным доступом из РФ:

| Категория | Выбранный сервис | Почему НЕ используем |
|-----------|-----------------|----------------------|
| Хостинг frontend | **Selectel** или **Timeweb Cloud** | Vercel периодически недоступен из РФ |
| База данных | **Supabase** (EU-West, Ireland) ИЛИ PostgreSQL на Selectel VPS | Supabase пока доступен, но иметь план Б |
| SMS | **СМС Центр (smsc.ru)** или **МТС Exolve** | Twilio заблокирован |
| Платежи | **ЮKassa** | Stripe недоступен в РФ |
| Push-уведомления | **Web Push (VAPID)** — свой сервер | Firebase Cloud Messaging нестабилен |
| Email | **Unisender** или **SMTP Mail.ru** | SendGrid/Mailgun блокируются |
| CDN/медиа | **Selectel Object Storage** или **VK Cloud S3** | AWS S3/Cloudflare нестабильны |
| Домен | Регистрация в **RU-CENTER** или **REG.RU** | — |

**Правило:** перед добавлением любого нового внешнего сервиса — проверить доступность из РФ.

## Стек технологий

### Frontend
- **Next.js 14** (App Router) — основной фреймворк
- **TypeScript** — строго, везде
- **Tailwind CSS** + **shadcn/ui** — UI компоненты
- **React Query (TanStack Query)** — серверный стейт и кэш
- **Zustand** — клиентский стейт
- **React Hook Form + Zod** — формы и валидация

### Backend
- **Next.js API Routes** — API
- **PostgreSQL** — база данных
- **Supabase** (self-hosted или Cloud EU) — auth, realtime, storage, RLS
- Резервный вариант БД: **PostgreSQL на Selectel VPS** + **Drizzle ORM**

### Хостинг (приоритет — российские площадки)
- **Selectel** — основной VPS/хостинг (Москва, СПб)
  - Альтернатива: **Timeweb Cloud**, **reg.ru**, **beget.com**
- Next.js деплоится как Docker-контейнер или через PM2 на VPS
- **Nginx** — reverse proxy перед Next.js
- **SSL** — Let's Encrypt через certbot

### Интеграции
- **ЮKassa** — оплата (единственный вариант для РФ)
- **smsc.ru** — SMS уведомления (российский оператор, работает стабильно)
- **Web Push VAPID** — push через свой сервер (без Firebase)
- **Unisender** или **Mail.ru SMTP** — email
- **Selectel Object Storage** — хранение файлов (фото студентов, документы)

## Роли пользователей

| Роль | Описание |
|------|----------|
| `student` | Клиент/студент/арендатор. Записывается, платит, получает уведомления |
| `teacher` | Преподаватель. Ведёт расписание, студентов, отмечает посещаемость |
| `admin` | Полный доступ. Видит всё: финансы, переносы, аналитику |

Роли хранятся через `profiles.role` в PostgreSQL + Row Level Security (RLS).

## Аутентификация — вход по SMS (не по email)

Основной способ входа — **номер телефона + SMS-код** (OTP).  
Email — опциональный, только для уведомлений.

**Почему SMS а не email:**
- Целевая аудитория — обычные люди, не IT-специалисты
- Телефон есть у всех, email помнят не все
- Меньше трений при регистрации

**Реализация:**
1. Пользователь вводит номер телефона (+7...)
2. Сервер через smsc.ru отправляет 4-значный код
3. Код хранится в Redis (или в БД) с TTL 5 минут
4. При правильном коде — создаётся сессия (JWT)
5. Supabase Auth Phone provider ИЛИ кастомная реализация

## Основные сущности системы

- **User / Profile** — пользователь (студент / преподаватель / админ)
- **Teacher** — профиль преподавателя
- **Student** — профиль студента (баланс минут, заметки)
- **LessonType** — тип занятия: вводное / групповое / индивидуальное / самопрактика / аренда
- **Course** — курс (пакет занятий с ценой)
- **Slot** — временной слот в расписании
- **Booking** — запись студента на слот
- **Purchase** — покупка курса/пакета
- **Payment** — платёж через ЮKassa
- **Notification** — очередь уведомлений (SMS / Push)
- **TransferRequest** — запрос преподавателя на перенос урока

## Ключевые пользовательские сценарии

### Студент
1. Регистрация через номер телефона (SMS-код)
2. Запись на вводный урок (1 раз, бесплатно или с предоплатой)
3. Запись на обычный урок (списывается с баланса или курса)
4. Покупка курса / пакета часов — оплата через ЮKassa
5. Запись на самостоятельную практику / аренду зала
6. Просмотр своих записей и истории
7. SMS за 24ч и за 2ч до урока (через smsc.ru)
8. Перенос или отмена записи

### Преподаватель
1. Вход по SMS
2. Настройка расписания доступности (шаблон по неделям)
3. Просмотр предстоящих уроков
4. Список своих студентов и их прогресс
5. Отметка посещаемости после урока
6. Запрос на перенос урока (идёт на подтверждение админу)

### Администратор
1. Дашборд: занятость залов, выручка, посещаемость
2. Управление расписанием всей школы
3. Управление переносами и отменами
4. Управление курсами и ценами
5. Управление пользователями (создание преподавателей)
6. Финансовые отчёты (ЮKassa + наличные)
7. Ручная рассылка SMS

## Правила разработки

1. **TypeScript strict mode** — никакого `any`
2. **RLS в PostgreSQL/Supabase** — безопасность на уровне БД
3. **Zod схемы** — валидация везде
4. **Транзакции** — любое изменение баланса/записи — только через транзакцию
5. **Идемпотентность** — `idempotency_key` в ЮKassa, дедупликация SMS
6. **Mobile-first** — сайт прежде всего для телефона
7. **Российские сервисы** — никаких зависимостей от заблокированных API
8. **Docker** — приложение упаковано в контейнер для деплоя на VPS
9. **Резервирование** — критичные данные (БД) с ежедневным бэкапом
